# Web Security Pre-Diagnostic (Junior Methodology)

This repository documents a generic methodology for performing a **non-intrusive web security pre-diagnostic**.

## Objective
Identify common security misconfigurations on a public website and propose **prioritized hardening recommendations**.

This work does **not** include:
- penetration testing
- vulnerability exploitation
- code review
- infrastructure audit

## Scope
- Public websites (B2B / B2C)
- Web security fundamentals
- Configuration and exposure analysis

## Methodology Overview
1. HTTPS / TLS verification
2. Security headers analysis
3. Browser permissions review
4. Cookie configuration checks
5. Public exposure and surface reduction
6. Risk prioritization

## Output
- Identified issues
- Risk level (low / medium)
- Clear remediation recommendations

## Disclaimer
All examples in this repository are **generic and fictional**.  
No real company, infrastructure or client data is used.
